## 2. 페이지별 상세 분석 및 문제점

### 2-1. 랜딩페이지 (`/`)

**현재 구성:** HeroSection → ServicesSection → WhySimpleCube (3개 섹션만)

**문제점:**
- ❌ **콘텐츠 절대 부족**: 서비스 소개 사이트인데 3개 섹션만으로 너무 단조로움
- ❌ **CTA(Call to Action) 없음**: 랜딩페이지에서 문의/예약으로 유도하는 장치가 전혀 없음
- ❌ **소셜 프루프 없음**: 고객 후기, 실제 사례, 수치적 신뢰도 부족
- ❌ **WhySimpleCube 수치 불일치**: "200+ 행사 진행"이라고 되어있는데, popup/ServiceIntro에서는 "1,600+ 연간 진행 건수"로 표기 → 수치 모순
- ❌ **WhySimpleCube "3개 전국 거점"**: 실제 푸터에는 4개 지점(서울, 부산, 대전, 전주) → 불일치
- ⚠️ **히어로 배경 이미지**: 1장(main.jpg)으로만 운영 중 → 슬라이드 또는 영상 고려 필요

**추가 필요 섹션 제안:**
1. 고객 후기/리뷰 섹션
2. 실제 포트폴리오 사진 하이라이트
3. 고객사 로고 배너 (실제 데이터)
4. 간편 문의 CTA 섹션
5. 인스타그램 피드 연동 또는 최신 사진 갤러리

### 2-2. 팝업 포토부스 (`/popup`)

**현재 구성:** PageHero → ServiceIntro → MachineShowcase → CustomOptions → PortfolioGallery → BookingProcess → ClientShowcase (7개 섹션)

**문제점:**
- ❌ **PortfolioGallery - 전부 Unsplash 외부 이미지**: 실제 포트폴리오가 아닌 stock 사진 6장 사용 중
  ```
  unsplash.com/photo-1511578314322 (이벤트 장면)
  unsplash.com/photo-1504384308090 (사무실)
  unsplash.com/photo-1497366858526 (도시)
  unsplash.com/photo-1560472354    (사무용품)
  unsplash.com/photo-1522202176988 (회의)
  unsplash.com/photo-1517457373958 (이벤트)
  ```
  → 실제 심플큐브 행사 사진으로 교체 필수
- ❌ **ClientShowcase - 완전한 플레이스홀더**: 'Client A' ~ 'Client F'로만 되어있음 → 실제 고객사명/로고 필요
- ⚠️ **SectionHeading 컴포넌트 미사용**: import는 하지만 실제로 사용하지 않는 컴포넌트가 여러 군데 있음 (PortfolioGallery, ClientShowcase)
- ⚠️ **카카오톡 링크 미확인**: `https://pf.kakao.com/_simplecube` → 실제 카카오 채널 URL인지 확인 필요
- ⚠️ **가격 정보 없음**: 비용 범위나 "견적 문의"에 대한 안내 부족

### 2-3. 웨딩 포토부스 (`/wedding`)

**현재 구성:** PageHero → ServiceIntro → PackageDetails → TemplateGallery(+Backdrop) → PartnerCompanies (5개 섹션)

**문제점:**
- ❌ **PartnerCompanies - 완전한 플레이스홀더**: 'Partner A' ~ 'Partner F'로만 되어있음 → 실제 파트너사명/로고 필요
- ⚠️ **가격 정보 부재**: 웨딩 포토부스 비용에 대한 정보가 전혀 없음 (범위라도 필요)
- ⚠️ **예약 가능일 정보 없음**: 예약 달력이나 가용성에 대한 안내 부족
- ⚠️ **후기/실제 사례 없음**: 실제 웨딩 촬영 결과물이나 후기 부재
- ⚠️ **추가 옵션 정보 부족**: 기본 구성 외 추가 가능한 옵션(영상 촬영, 추가 인화 등)에 대한 설명 부족

### 2-4. 브랜드 페이지 (`/brand`)

**현재 구성:** PageHero → BrandStory → BranchOffices (3개 섹션)

**문제점: (가장 문제가 많은 페이지)**
- ❌ **PageHero 배경 이미지 - Unsplash 외부 이미지**: 실제 회사 사진이 아닌 stock 사진 사용
  ```
  unsplash.com/photo-1497366216548 (일반 사무실)
  ```
- ❌ **BrandStory 이미지 - 전부 Unsplash**: 2개 이미지 모두 stock 사진
  ```
  unsplash.com/photo-1497366216548 (사무실 - 히어로와 동일)
  unsplash.com/photo-1522202176988 (회의장면 - popup 포트폴리오와 중복!)
  ```
- ❌ **BranchOffices - 데이터 불일치 심각**:
  - 컴포넌트: 서울 본사, 부산 지사, **제주 지사** (3개)
  - 푸터: 서울 본점, 부산 지점, **대전 지점, 전주 지점** (4개)
  - **제주 지사가 브랜드 페이지에만 존재**, **대전/전주는 푸터에만 존재**
- ❌ **BranchOffices 연락처 - 가짜 데이터**:
  ```
  서울: 02-1234-5678 (가짜)
  부산: 051-1234-5678 (가짜)
  제주: 064-1234-5678 (가짜)
  ```
  실제 푸터 연락처와 완전히 다름:
  ```
  서울: 02-338-9180
  부산: 010-3122-4746
  대전: 010-8838-8122
  전주: 010-5743-8122
  ```
- ❌ **BranchOffices 이미지 - Unsplash**: 3개 지점 모두 stock 사진
- ❌ **BranchOffices 주소 - 부정확**: "서울특별시 강남구", "부산광역시 해운대구" 등 구체적 주소 없음
- ❌ **회사 설립 연도 불일치**: BrandStory에서는 "2020년 시작", popup/ServiceIntro에서는 "Since 2019" → 어느 것이 맞는지 확인 필요
- ⚠️ **회사 연혁/타임라인 없음**: 설립부터 현재까지의 히스토리 부재
- ⚠️ **팀 소개 없음**: 구성원 소개 부재
- ⚠️ **비전/미션 스테이트먼트**: 추상적으로만 언급, 구체적 비전 부재

### 2-5. FAQ 페이지 (`/faq`)

**현재 구성:** 헤더 → AccordionList (8개 문항) → 연락 CTA

**문제점:**
- ❌ **연락처 불일치**: FAQ 하단 CTA에 `hello@simplecube.net`과 Instagram `simplecube`로 되어있으나:
  - 푸터에서는 `simple_cube@naver.com` 사용
  - 인스타그램은 `simplecube_photobooth` 사용
  → 어느 것이 공식인지 통일 필요
- ⚠️ **FAQ 항목 부족**: 8개 문항으로 주요 내용만 커버. 추가 필요한 질문들:
  - 촬영 시간은 얼마나 되나요?
  - 인화 사진 사이즈는?
  - 야외 행사도 가능한가요?
  - 디지털 파일은 언제 받을 수 있나요?
  - 소품은 제공되나요?
  - 행사 몇 시간 전에 세팅하나요?
- ⚠️ **카테고리 필터 없음**: category 데이터는 있지만 UI에서 카테고리별 필터 기능 없음
- ⚠️ **검색 기능 없음**: FAQ 검색 지원 안됨

## 3. 콘텐츠 부족/오류 사항

### 3-1. 이미지 관련

| 위치 | 문제 | 심각도 |
|------|------|--------|
| Brand PageHero 배경 | Unsplash stock 사진 | 🔴 높음 |
| BrandStory 이미지 2개 | Unsplash stock 사진 | 🔴 높음 |
| BranchOffices 이미지 3개 | Unsplash stock 사진 | 🔴 높음 |
| Popup PortfolioGallery 6개 | Unsplash stock 사진 | 🔴 높음 |
| Blog 대표 이미지 3개 | Unsplash stock 사진 | 🟡 중간 |
| Brand 페이지 히어로 ↔ BrandStory | 동일 이미지 중복 사용 | 🟡 중간 |
| BrandStory ↔ Popup Portfolio | 동일 이미지 중복 사용 | 🟡 중간 |

**필요한 실제 이미지 목록:**
1. 회사/사무실/작업 현장 사진 (Brand 페이지용) - 최소 3장
2. 각 지점 실제 사진 (BranchOffices용) - 지점당 1장
3. 실제 팝업 행사 현장 포트폴리오 사진 - 최소 6~12장
4. 팀/직원 사진 (선택)
5. 블로그용 실제 사진

### 3-2. 텍스트/데이터 관련

## 4. 디자인/UX 개선 필요 사항

### 4-1. 전체 공통

- ⚠️ **로고 없음**: 텍스트로만 "SIMPLE CUBE" 표시, 실제 로고 이미지 미사용
- ⚠️ **파비콘**: 기본 Astro 파비콘(favicon.svg) 사용 중 → 브랜드 파비콘 필요
- ⚠️ **OG 이미지 없음**: BaseLayout에서 `ogImage` prop이 있지만 기본값이 빈 문자열 → SNS 공유 시 이미지 없음
- ⚠️ **모바일 메뉴 닫기 버튼**: 데스크톱 → 모바일 전환 시 체크 필요
- ⚠️ **페이지 전환 애니메이션 없음**: 페이지 이동 시 즉시 전환만 됨

### 4-2. 네비게이션

- ⚠️ **현재 페이지 표시 부족**: 어떤 페이지에 있는지 시각적 구분이 약함
- ⚠️ **CTA 버튼 없음**: 네비게이션에 "문의하기" 같은 강조 버튼이 없음
- ⚠️ **전화번호/카카오톡 바로 연결 없음**: 헤더에 빠른 연락처 접근 수단 부재

### 4-3. 푸터

- ⚠️ **카카오톡 링크 없음**: 카카오 채널 ID만 텍스트로 표기, 클릭 가능한 링크가 아님
- ⚠️ **인스타그램 링크 없음**: 인스타 계정명만 텍스트로 표기, 클릭 가능한 링크가 아님
- ⚠️ **전화번호 링크 없음**: 전화번호가 텍스트로만 표기, `tel:` 링크가 없어 모바일에서 바로 전화 불가
- ⚠️ **저작권 표시 없음**: 기존 저작권 정보가 푸터 재작업 시 제거됨
- ⚠️ **개인정보처리방침/이용약관 링크 없음**

### 4-4. 각 페이지

- ⚠️ **CTA 동선 부족**: 각 페이지 하단에서 다음 행동(문의, 다른 서비스 보기 등)으로 유도하는 장치 미비
- ⚠️ **뒤로가기/이전 페이지 내비게이션**: 서브 페이지에서 홈으로 돌아가는 동선이 네비게이션뿐
- ⚠️ **스크롤 진행률 표시**: 긴 페이지(popup, wedding)에서 현재 위치 파악 어려움

## 5. 기술적 문제 및 최적화

### 5-1. 성능

- ⚠️ **외부 이미지 의존**: Unsplash 이미지를 직접 참조 → CDN 장애 시 이미지 깨짐, 속도 저하
- ⚠️ **이미지 최적화 미적용**: Astro의 `<Image>` 컴포넌트 미사용 → WebP 변환, 반응형 사이즈 미적용
- ⚠️ **폰트 CDN 의존**: Pretendard 폰트를 CDN에서 로드 → 로컬 호스팅 고려
- ⚠️ **GSAP 번들 사이즈**: GSAP + ScrollTrigger를 전체 페이지에 로드

### 5-2. 코드 품질

- ⚠️ **SectionHeading 컴포넌트 미사용**: 5개 이상의 컴포넌트에서 import 후 사용하지 않음
- ⚠️ **인라인 SVG 반복**: 여러 컴포넌트에서 동일한 SVG 아이콘을 인라인으로 반복 사용
- ⚠️ **일관성 없는 섹션 헤딩 패턴**: SectionHeading 컴포넌트가 있지만 대부분 인라인 마크업 사용

### 5-3. 접근성 (Accessibility)

- ⚠️ **alt 텍스트 개선 필요**: 일부 이미지 alt가 "팝업 포트폴리오 1" 등 설명이 불충분
- ⚠️ **키보드 네비게이션**: FAQ 아코디언의 키보드 접근성 확인 필요
- ⚠️ **색상 대비**: 라이트 테마 전환 후 `text-secondary`(#6e6e73)의 대비율 확인 필요
- ⚠️ **aria-label 누락**: 일부 인터랙티브 요소에 적절한 레이블 없음

## 6. SEO 및 마케팅 관점 분석

### 6-1. 메타 데이터

- ⚠️ **OG Image 미설정**: 모든 페이지에 og:image가 비어있음 → SNS 공유 시 미리보기 없음
- ⚠️ **구조화 데이터 부족**: 블로그만 JSON-LD 있고, 서비스 페이지에는 LocalBusiness 등의 구조화 데이터 없음
- ⚠️ **canonical URL**: 설정은 되어있으나 GitHub Pages URL 기준 → 커스텀 도메인 시 변경 필요

### 6-2. 콘텐츠 SEO

- ❌ **키워드 전략 부재**: "포토부스 대여", "웨딩 포토부스", "팝업 포토부스" 등 핵심 키워드에 대한 체계적 최적화 없음
- ❌ **지역 SEO 부재**: "서울 포토부스", "부산 포토부스" 등 지역 키워드 타겟 없음
- ⚠️ **블로그 글 부족**: 3개로는 검색 유입 효과 미미
- ⚠️ **내부 링크 전략 부재**: 페이지 간 상호 연결이 약함

### 6-3. 전환(Conversion) 최적화

- ❌ **문의 폼 없음**: 전체 사이트에 문의 양식이 단 하나도 없음
- ❌ **전화번호 클릭 불가**: 모바일에서 바로 전화할 수 없음
- ❌ **카카오톡 채널 연결 미흡**: 카카오 채널 버튼이 특정 섹션에만 있음
- ⚠️ **플로팅 CTA 없음**: 스크롤 시에도 접근 가능한 문의 버튼 없음
